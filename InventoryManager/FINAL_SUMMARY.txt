KEY FEATURES
================================================================================

LIFECYCLE STATUS VALUES:
  • active         - Device in service (counted in statistics)
  • sold           - Permanently removed from inventory (excluded)
  • decommissioned - End-of-life/obsolete (excluded)
  • damaged        - Broken/needs repair (excluded)
  • retired        - Voluntarily removed (excluded)

DEVICE MODEL METHODS (11):
  • is_in_service() - Check if device is active and online
  • is_excluded_from_statistics() - Check exclusion status
  • can_create_new_rental() - Check if rental allowed
  • mark_as_sold(reason) - Mark device as sold
  • mark_as_decommissioned(reason) - Mark as stopped use
  • mark_as_damaged(reason) - Mark as broken
  • mark_as_retired(reason) - Mark as retired
  • restore_to_active(reason) - Restore from any status
  • set_lifecycle_status(new_status, reason) - Generic setter
  • get_device_count_by_lifecycle_status() - Statistics query
  • get_active_devices() - Filter for active devices
  • get_excluded_devices() - Filter for excluded devices

REST API ENDPOINTS (4):
  • PUT /api/devices/<id>/lifecycle - Update lifecycle status
  • PUT /api/devices/<id>/mark-sold - Quick-mark as sold
  • GET /api/devices/lifecycle/summary - Lifecycle statistics
  • GET /api/devices/lifecycle/list?status=<status> - Filter by status
