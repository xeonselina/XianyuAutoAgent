# 客户历史订单查询 & 镜头组合需求文档

> 创建时间：2026-06-22
> 状态：需求评审中

## 一、背景与目标

### 1.1 现状
- 甘特图是日常运营核心入口。工具栏聚集了「添加设备 / 预定设备 / 批量发货 / 更多 / 刷新」等操作。
- 报价时，运营需要快速了解客户的历史成交价、租期长度、机型/镜头偏好，从而判断本次报价的合理区间。当前没有「按客户聚合」的视图，只能依赖 `搜索租赁人名/地址` 这种线性方式。
- 当前所有租赁数据模型 `rentals` 假定「默认套装 = 主机 + 增距镜 + 配套配件」，但实际镜头组合差异很大：
  - **x300u（300Ultra）** 有 **400MM 镜头（默认）/ 200MM 镜头 / 裸机 / 双镜头** 四种组合。
  - **x200u（200Ultra）/ x300pro（300Pro）** 有 **200MM 镜头 / 裸机** 两种组合。
- 现有打印发货单 `templates/shipping_order2.html` 与热敏面单 `shipping_slip_image_service.py` 的品名清单是写死的（VIVO X200 Ultra + 90w 充电头 + 2.35x 增距镜 + 便携手提包），**没有按机型和镜头组合动态渲染**。

### 1.2 目标
1. 甘特图上提供一键查询某个客户最近 5 单的入口，按日期倒排，展示：机型、镜头组合、租赁价格、起止时间、租赁天数、日均价。
2. 让租赁记录支持选择「镜头组合」：
   - x300u 4 选 1（400MM / 200MM / 裸机 / 双镜头），默认 400MM。
   - x200u、x300pro 2 选 1（200MM / 裸机），默认 200MM。
3. PC 和移动端的租赁表单都支持镜头组合选择，做响应式适配。
4. 打印发货单（HTML）和热敏面单（图像）根据所选镜头组合，动态渲染品名清单。

### 1.3 非目标
- 不做客户档案的 CRUD（不引入新的 `customers` 表）。客户仍然以 `customer_name + customer_phone + buyer_id` 现有字段聚合识别。
- 不做「镜头组合模板配置中心」。组合定义以枚举值写死，后续如需扩展再迭代。
- 不做历史数据的回填迁移，旧数据 `lens_combo` 字段默认为各机型的默认组合（详见下文）。

## 二、功能需求

### 2.1 客户历史订单查询

#### 2.1.1 入口
- **PC 甘特图（`frontend/src/components/GanttChart.vue`）工具栏**：在「更多」下拉之前新增 `客户历史` 按钮。
- **移动端甘特图（`frontend-mobile/src/views/GanttView.vue`）**：在顶栏/筛选区附近加入「客户历史」入口（图标 + 文字）。

#### 2.1.2 查询流程（模糊搜索 + 多结果选择）
1. 点按钮 → 弹出查询面板（PC 用 `el-dialog`，移动端用 `van-popup` 全屏）。
2. 输入框支持「电话 / 姓名 / 闲鱼 ID」**模糊**输入，键入后防抖 300ms 触发搜索（回车立即查询）。
3. 后端按以下规则聚合候选：
   - 模糊匹配 `customer_phone`（contains，数字归一化后比较）
   - 模糊匹配 `customer_name`（contains）
   - 模糊匹配 `buyer_id`（contains）
4. 候选列表每条显示：「`姓名` · `电话末 4 位` · `闲鱼 ID` · `历史单数`」，按总单数倒序。最多 20 条。
5. 点击某条候选 → 切换到「该客户历史 5 单」明细视图（同一弹窗内，提供「返回搜索」按钮）。

#### 2.1.3 历史 5 单明细
- 取该客户**最近 5 单**，**按 `start_date` 倒序**（同日按 `created_at` 倒序）。
- 每行展示字段：
  | 字段 | 来源 |
  |---|---|
  | 机型 + 镜头组合 | `device_model.display_name` + 中文化 `lens_combo`，如「300Ultra · 200MM」「200Ultra · 裸机」 |
  | 租赁价格 | `rental.order_amount`（¥） |
  | 开始日期 | `rental.start_date` |
  | 结束日期 | `rental.end_date` |
  | 租赁天数 | `end_date - start_date + 1`（复用 `Rental.get_duration_days()`） |
  | 日均价 | `order_amount / 天数`（前端计算，便于报价参考；高亮） |
- 视觉：PC 表格、移动端卡片列表。
- 该客户少于 5 单时显示实际单数；无单数显示空态。

#### 2.1.4 客户聚合规则
- **以归一化 `customer_phone`（去非数字字符）为主键**聚合订单。
- phone 为空时，退化为 `customer_name + buyer_id` 联合键。
- 同名同电话但 `buyer_id` 不同视为同一客户（电话最稳定）。
- 候选列表展示的是聚合后的「客户行」；点进去后展示该客户名下所有订单（取最新 5 条）。

### 2.2 镜头组合

#### 2.2.1 数据模型
在 `rentals` 表新增字段 **`lens_combo`**（命名：lens combination）：

```python
lens_combo = db.Column(
    db.Enum('lens_400mm', 'lens_200mm', 'bare', 'lens_dual',
            name='rental_lens_combo'),
    nullable=False,
    server_default='lens_400mm',  # 由迁移脚本按机型回填
    comment='镜头组合: lens_400mm=400MM镜头 / lens_200mm=200MM镜头 / bare=裸机 / lens_dual=双镜头(仅x300u)'
)
```

#### 2.2.2 各型号可选项与默认值
| 型号 | 可选组合 | 默认 |
|---|---|---|
| **x300u（300Ultra）** | `lens_400mm` / `lens_200mm` / `bare` / `lens_dual` | `lens_400mm` |
| **x200u（200Ultra）** | `lens_200mm` / `bare` | `lens_200mm` |
| **x300pro（300Pro）** | `lens_200mm` / `bare` | `lens_200mm` |

**校验规则（后端）**：
- 写入时按上表校验「型号-镜头组合」组合合法性，非法返回 400。
- 切换机型导致原组合非法时，前端必须回退到该机型默认值。

#### 2.2.3 历史数据迁移
- 新增 Alembic migration `add_lens_combo_to_rentals`：
  - 添加列（先 nullable），按当前 `device.model` 回填默认值：x300u → `lens_400mm`，x200u/x300pro → `lens_200mm`。
  - 回填完成后改为 NOT NULL。

#### 2.2.4 租赁表单 UI

**PC（`frontend/src/components/rental/RentalBasicForm.vue` 或新增 `LensComboSelector.vue` 子组件）**：
- 在「设备选择」下方新增「镜头组合」label，控件为 `el-radio-group` 单选。
- 选项随当前机型动态变化（x300u 4 项，x200u/x300pro 2 项）。
- 切换机型时若 `lens_combo` 不在新机型可选集中 → 回退为新机型默认值。

**移动端（`frontend-mobile/src/views/CreateRentalView.vue` / `EditRentalView.vue`）**：
- 使用 `van-radio-group` + chip 样式（圆角 tag），与既有附件选择视觉一致。
- 4 个选项时自动换行；保证 iPad 与手机视口都不挤压。

**响应式约束**（遵循 `InventoryManager/CLAUDE.md`）：
- ≥ 1024px：横排单选按钮。
- 768–1023px：横排，字号缩小。
- < 768px：折叠为 chip 网格，可换行。

#### 2.2.5 打印发货单 & 热敏面单

新增工具 `app/services/printing/rental_product_lines.py`：

```python
def get_product_lines(rental):
    """根据机型 + 镜头组合返回品名清单。"""
    model = rental.device.device_model.name  # 'x200u' / 'x300pro' / 'x300u'
    combo = rental.lens_combo

    lines = [
        {'name': model_display(model) + ' 主机', 'qty': 1, 'is_main': True},
        {'name': '90w 充电头+充电线', 'qty': 1},
    ]
    if combo == 'lens_400mm':
        lines.append({'name': '400MM 增距镜+增距镜脚架+手机壳', 'qty': 1})
    elif combo == 'lens_200mm':
        lines.append({'name': '200MM 镜头+手机壳', 'qty': 1})
    elif combo == 'lens_dual':
        lines.append({'name': '400MM 增距镜+增距镜脚架+手机壳', 'qty': 1})
        lines.append({'name': '200MM 镜头', 'qty': 1})
    # bare: 不追加镜头行

    if combo != 'bare':
        lines.append({'name': '套装便携手提包', 'qty': 1})
    return lines
```

- **HTML 发货单 `templates/shipping_order2.html`**：把写死的 4 行 `<tr>` 改为 `{% for item in product_lines %} ... {% endfor %}`。`_prepare_shipping_order_data` 注入 `product_lines`。
- **热敏图发货单 `shipping_slip_image_service.py`**：当前 `_draw_info_row(draw, y, "设备:", device_name)` 单行；改为遍历 `product_lines`，逐行画 `_draw_info_row(draw, y, "品名{n}:", line['name'])`，主行高亮。

#### 2.2.6 客户历史明细中展示镜头组合
2.1.3 的明细将「机型」列升级为「机型 + 镜头组合」中文形式，便于报价时一眼判断（如「300Ultra · 双镜头」）。

## 三、API 设计

### 3.1 新增端点（蓝图：`app/routes/customer_api.py`）
| Method | Path | 说明 |
|---|---|---|
| GET | `/api/customers/search?q=xxx` | 模糊聚合搜索；返回 `[{customer_name, customer_phone_masked, buyer_id, total_rentals}]`，限 20 条 |
| GET | `/api/customers/rentals?phone=&name=&buyer_id=&limit=5` | 取指定客户最近 N 单，按 `start_date desc, created_at desc`；返回精简字段 |

### 3.2 现有端点变更
- `POST/PUT /api/rentals`：请求体接受 `lens_combo`；后端校验「型号 × 镜头组合」合法性。
- `Rental.to_dict()`：返回 `lens_combo`（甘特图、tooltip、编辑回显都依赖）。
- 甘特图数据接口透传 `lens_combo`。

## 四、数据库迁移
- `migrations/versions/<new>_add_lens_combo_to_rentals.py`：
  1. `add_column('rentals', lens_combo Enum nullable=True)`
  2. 数据回填：x300u → `lens_400mm`；x200u / x300pro → `lens_200mm`。
  3. Alter column → NOT NULL，server_default = `lens_400mm`（占位；语义由代码层保证）。
  4. 顺带补 `customer_phone`、`customer_name` 索引（如缺失）。
- Downgrade：`drop_column`。

## 五、验收标准
1. 甘特图工具栏「客户历史」按钮：模糊输入电话/姓名 → 候选列表 → 选客户 → 5 单明细，含日均价；PC 与移动端都能用，移动端弹窗不被键盘遮挡，候选可滚动。
2. 创建租赁：
   - 选 x300u → 镜头组合 4 项可选，默认 400MM。
   - 选 x200u 或 x300pro → 镜头组合 2 项（200MM / 裸机），默认 200MM。
   - 切换机型导致原组合非法 → 自动回退默认值。
   - 调 API 手工传非法组合（如 x200u + lens_dual）→ 400。
3. 打印发货单（HTML）与热敏面单（图像）按镜头组合渲染：
   - x300u + lens_400mm：主机/充电头/400MM 增距镜+脚架+手机壳/便携包。
   - x300u + lens_200mm：主机/充电头/200MM 镜头+手机壳/便携包。
   - x300u + lens_dual：主机/充电头/400MM 增距镜+脚架+手机壳/200MM 镜头/便携包（5 行）。
   - 任意机型 + bare：仅主机 + 充电头（2 行）。
   - 旧订单（迁移后回填值）：与改动前的人工识别一致。
4. 甘特图 tooltip 与客户历史明细均显示中文化镜头组合（如「300Ultra · 双镜头」）。
5. PC、iPad、手机三种宽度下 UI 不错位、按钮可点。
6. 现有甘特图、批量发货、合同等功能不受影响（回归通过）。

## 六、非功能要求
- 新代码遵循项目现有 Blueprint + handler 模式（参考 Phase 2 refactor）。
- `/api/customers/search` 性能依赖 `rentals.customer_phone`、`rentals.customer_name` 索引（迁移脚本检查并补齐）。
- 新增 E2E 测试覆盖核心路径，参考 `frontend-mobile/e2e/accessory-selection.spec.ts`。
