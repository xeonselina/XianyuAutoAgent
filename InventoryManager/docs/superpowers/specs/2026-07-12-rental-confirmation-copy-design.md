# Rental 保存后客户确认信息设计

## 目标

每次成功新建或编辑 rental 后，弹出一份来自数据库最终记录的客户确认信息。信息按关键字段逐行展示，可一键复制到闲鱼聊天框供客户确认。

## 触发范围

- `BookingDialog` 新建 rental 成功后触发。
- `EditRentalDialogNew` 编辑 rental 成功后触发。
- 只有保存接口成功才触发；保存失败沿用现有错误提示，不显示确认弹窗。
- 保存成功后先关闭新建或编辑表单，再显示确认弹窗。

## 数据来源与流程

采用“保存后重新读取”方案：

1. 新建成功时，从创建接口响应取得主 rental ID；编辑成功时使用当前 rental ID。
2. 新建和编辑组件通过 `success` 事件把 rental ID 传给 `GanttChart`。
3. `GanttChart` 完成现有数据刷新后，调用 `getRentalById(rentalId)` 读取数据库中的最终 rental。
4. 查询成功后，把完整 rental 交给共用的 `RentalConfirmationDialog`。
5. 查询失败时不使用表单临时值或旧 rental 兜底，提示“保存成功，但确认信息加载失败”，避免向客户复制不准确的信息。

此功能不修改数据库结构，也不改变后端保存接口的业务行为。

## 组件边界

### `RentalConfirmationDialog.vue`

- 接收完整 rental 和显示状态。
- 展示格式化后的五行确认信息。
- 提供“复制全部”和“关闭”按钮。
- 正文允许用户手动选择。
- 复制成功后弹窗保持打开，并提示“确认信息已复制”。

### `rentalConfirmation.ts`

提供纯函数，负责把 rental 转换为结构化行和最终复制文本。该模块不访问 Pinia、接口或浏览器剪贴板，便于完整测试字段规则。

### `GanttChart.vue`

统一承接新建和编辑成功事件，按 rental ID 重新查询，并控制唯一的确认弹窗实例。新建和编辑表单不各自维护一套确认文案。

## 确认文案

复制内容固定为五行，不添加标题、空行或额外说明：

```text
收货地址：张三，13800138000，广东省广州市天河区……
寄出时间：2026-07-12
预计收货：2026-07-14
客户归还：2026-07-20
寄出型号：VIVO X300 Ultra + 双镜头 + 镜头支架 + 手柄 + 手机支架
```

字段对应关系：

| 展示字段 | 数据来源 | 规则 |
| --- | --- | --- |
| 收货地址 | `destination`、`customer_phone` | 以收件信息为主；单独填写的客户电话不在收件信息中时自动追加；不使用闲鱼 ID 作为联系人 |
| 寄出时间 | `ship_out_time` | 只显示 `YYYY-MM-DD` |
| 预计收货 | `start_date` | 只显示 `YYYY-MM-DD` |
| 客户归还 | `end_date` | 只显示 `YYYY-MM-DD` |
| 寄出型号 | `device`、`lens_combo`、附件字段 | 用 ` + ` 连接型号、镜头组合及实际附件 |

不显示 `ship_in_time`、设备编号、设备序列号、闲鱼 ID 或代传照片信息。

## 型号、镜头和附件规则

- 型号优先使用 `device.device_model.display_name`；没有时依次使用 `device.device_model.name`、`device.model`，最后显示“未识别型号”。
- 镜头组合复用现有 `lensComboDisplay` 中文映射：`400MM 镜头`、`200MM 镜头`、`裸机`、`双镜头`；缺失时显示“未填写镜头组合”。
- 配套附件：`includes_lens_mount` 显示“镜头支架”，`includes_handle` 显示“手柄”。
- 库存附件根据 `accessories` 中的类型或名称显示“手机支架”“三脚架”；其他附件使用自身名称或型号。
- 附件名称去重，只列实际包含的附件，不列设备 ID 或序列号。
- 没有任何附件时，在型号行末尾显示“无附件”。

示例：

```text
寄出型号：VIVO X300 Ultra + 400MM 镜头 + 无附件
```

## 缺失值规则

- 收件信息为空时显示 `收货地址：未填写`；若只有客户电话，则显示该电话。
- 任一时间为空或无法解析时，对应行显示“未填写”，但不省略该行。
- 型号和镜头按上一节的明确兜底文案显示。

## 复制兼容性

1. 优先调用 `navigator.clipboard.writeText`。
2. NAS 通过 HTTP IP 访问时，Clipboard API 可能不可用；此时自动创建临时 `textarea`，选择完整文本并调用兼容复制命令。
3. 两种方式都失败时提示“自动复制失败，请手动选择文本复制”，弹窗正文保持可选择。

## 错误与状态处理

- 保存期间沿用现有 loading 和重复提交保护。
- 保存成功后详情查询期间不重复显示确认弹窗。
- 查询成功才设置确认 rental 并打开弹窗。
- 查询返回空值或抛错时清空确认 rental，保持确认弹窗关闭，并显示加载失败提示。
- 关闭确认弹窗只清理当前展示状态，不影响已经保存的 rental。

## 测试策略

### 纯函数单元测试

- 收件信息自动补充单独电话，已有相同电话时不重复。
- 三个时间均只保留日期，缺失或无效值显示“未填写”。
- 型号优先级和镜头中文映射正确。
- 手柄、镜头支架、手机支架、三脚架和其他附件正确汇总并去重。
- 无附件时显示“无附件”。
- 最终文本严格为五行，不包含预计收回时间和闲鱼 ID。

### 组件测试

- 确认弹窗逐行展示生成内容。
- Clipboard API 成功时写入完整文本并显示成功提示。
- Clipboard API 不可用或失败时使用兼容复制。
- 两种复制方式都失败时提示手动复制，正文仍存在。

### 流程测试

- 新建成功事件携带主 rental ID。
- 编辑成功事件携带当前 rental ID。
- `GanttChart` 收到 ID 后重新读取 rental 并打开确认弹窗。
- 详情查询失败时不打开确认弹窗，并显示明确提示。
- 保存失败时不查询详情、不打开确认弹窗。
