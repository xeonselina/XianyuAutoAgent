# 甘特图求解状态中文化设计

## 目标

在“一键重排档期”的预览表格中使用中文展示 OR-Tools 求解状态，让用户可以直接理解方案是否可执行、是否已证明最优。

## 范围

仅修改前端展示。后端 API 继续返回现有英文状态码，执行按钮的可用性判断也继续基于 `OPTIMAL` 和 `FEASIBLE`，避免改变接口契约和业务逻辑。

状态映射如下：

| 后端状态码 | 前端显示 |
| --- | --- |
| `OPTIMAL` | 最优方案 |
| `FEASIBLE` | 可行方案 |
| `INFEASIBLE` | 无可行方案 |
| `UNKNOWN` | 未得出结果 |
| `MODEL_INVALID` | 求解模型无效 |

未识别的状态码显示“未知状态”，不直接暴露英文内部值。

## 实现

在 `ScheduleReorderDialog.vue` 中增加一个纯展示用的状态格式化函数，并将“求解状态”列改为使用插槽调用该函数。原始预览对象保持不变。

## 验证

更新 `ScheduleReorderDialog.spec.ts`：断言 `OPTIMAL` 显示为“最优方案”，且页面不再显示原始英文状态；同时为其余已知状态和未知状态补充映射单元断言。现有执行流程测试继续验证英文状态码仍能正确启用执行按钮。
