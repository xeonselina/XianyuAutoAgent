# 租赁状态说明

## 租赁状态定义

### 1. `pending` - 待审批
- **含义**: 租赁申请已提交，等待审批
- **说明**: 客户提交租赁申请后，系统自动设置为待审批状态
- **可执行操作**: 审批通过、拒绝、取消

### 2. `active` - 租赁中
- **含义**: 租赁申请已审批通过，设备正在被租赁使用
- **说明**: 设备已交付给客户，处于租赁期间
- **可执行操作**: 完成租赁、延期、取消（在允许范围内）

### 3. `completed` - 已完成
- **含义**: 租赁已正常完成，设备已归还
- **说明**: 客户按时归还设备，租赁关系结束
- **可执行操作**: 无（终态）

### 4. `cancelled` - 已取消
- **含义**: 租赁申请或租赁被取消
- **说明**: 可能的原因包括客户主动取消、审批拒绝、或其他原因
- **可执行操作**: 无（终态）

### 5. `overdue` - 逾期
- **含义**: 租赁已超过约定归还日期，设备未归还
- **说明**: 客户未按时归还设备，需要及时处理
- **可执行操作**: 归还设备（转为completed）、延期、联系客户

## 状态流转图

```
pending → active → completed
   ↓         ↓        ↑
cancelled  overdue ───┘
```

## 状态转换规则

1. **pending → active**: 审批通过
2. **pending → cancelled**: 审批拒绝或主动取消
3. **active → completed**: 正常归还
4. **active → overdue**: 超过归还日期
5. **active → cancelled**: 租赁期间取消
6. **overdue → completed**: 逾期归还
7. **overdue → active**: 延期处理

## 注意事项

- 只有 `pending` 状态的租赁可以审批
- 只有 `active` 状态的租赁可以延期
- `completed` 和 `cancelled` 是终态，不能再次修改
- `overdue` 状态需要特别关注，及时处理
