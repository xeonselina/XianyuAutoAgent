# 2026-03-27

## 多模型降本改造：轻量任务使用 qwen3.5-flash

实施了"配置式多模型"方案，将非核心推理的 LLM 调用从 `qwen-plus/qwen3.5-plus` 切换到 `qwen3.5-flash`。

### 修改的文件
1. **`config/settings.py`** - 新增 `model_name_light` 配置项，默认值 `qwen3.5-flash`
2. **`llm/qwen_client.py`** - `call_qwen()` 新增 `model` 参数，支持在调用点覆盖默认模型
3. **`agent/turn.py`** - 置信度评估 (`_estimate_response_confidence_percent`) 改用 `model_name_light`
4. **`agent/context_summarizer.py`** - 会话内摘要 (`summarize_context`) 改用 `model_name_light`
5. **`agent/executor.py`** - 跨会话历史摘要 (`_summarize_history`) 改用 `model_name_light`
6. **`services/sentiment_service.py`** - 情感三分类 (`analyze_sentiment`) 改用 `model_name_light`
7. **`.env.example`** - 新增 `MODEL_NAME_LIGHT=qwen3.5-flash` 配置示例

### 架构决策
- 核心 turn（agent 推理 + function calling）保持使用 `model_name`（plus 模型）
- 轻量任务（评分/摘要/分类）使用 `model_name_light`（flash 模型）
- 不采用"网关路由"方案，因为场景数量固定，硬编码选择模型更简单可靠
