## ai_kefu: 基于分析报告的头部问题逐一优化

### 修改的文件和具体改动

**1. `prompts/rental_system_prompt.py` — 彻底重写 System Prompt**
- 核心变化：从 271 行正式 Prompt → 91 行口语化 Prompt
- **强制 50 字限制**：最高优先级规则，每条回复不超过 50 字
- **学习人工风格**：给出真实人工回复范例（如"好的"、"什么时候用呢"）
- **用「你」不用「您」**：匹配闲鱼 C2C 平台调性
- **禁止编造**：明确禁止编造电话/微信/邮箱/价格/政策
- **追问上限 2 次**：同一信息最多问 2 次，然后等用户主动说
- **简短场景规则**：道别/确认/表情等场景 ≤10 字
- **卡片消息处理**：尝试理解卡片内容，不再说"请用文字描述"
- **议价处理规则**：先查价格再回应，不说"无法修改"
- **简化工作流程**：6 个阶段→直接一个简洁的步骤列表
- 更新 RENTAL_QUICK_REPLIES 为口语化风格

**2. `tools/parse_date.py` — 增强日期解析**
新增支持格式：
- 点号分隔：`4.15` → 4月15日
- 日期范围：`4.30-5.4` → 返回 date + end_date
- 中文范围：`4月30号到5月5号`
- 相对星期：`这周六`、`下周一`
- 带动作后缀：`15号收`、`4.20寄回`
- 大后天、斜杠格式(4/15)
- 重构为 `_resolve_date()` 辅助函数减少重复

**3. `config/settings.py` — 调整 LLM 参数**
- `qwen_temperature`: 0.7 → **0.3**（更稳定、减少废话）
- `qwen_max_tokens`: 2048 → **256**（从根本上限制回复长度）

**4. `scripts/init_rental_knowledge.py` — 新增 3 条知识**
- `rental_008`: 议价与优惠策略（砍价应对、多台优惠、不能做的事）
- `rental_009`: 已下单后处理流程（确认信息、发货、收货、归还）
- `rental_010`: 演唱会场景FAQ（提前收货、配件说明、拍摄建议、多人拼租）

### 后端改动

**1. 数据库迁移**
- `migrations/003_create_system_prompts_table.sql`：创建 `system_prompts` 表（id, prompt_key, title, content, description, active, timestamps），支持多版本、活跃版本标记

**5. API 注册**
