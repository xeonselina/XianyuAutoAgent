## ai_kefu: conversations 表 agent_response 为空问题修复
**根因分析**：
1. HTTP client 的 `agent_timeout` 只有 10 秒，但 Agent 多轮执行（LLM + tool calls）实际耗时 30+ 秒
2. ReadTimeout → tenacity 重试 3 次（每次 Agent 从头执行）→ 全部超时 → RetryError
3. Agent 在后台实际完成了工作（agent_turns 有数据），但 HTTP 连接已断开
4. 异常处理中直接 return None，`_log_message()` 未被调用 → conversations.agent_response 为空

**修复内容**：
1. `config.py` + `models.py`: agent_timeout 10s → 120s, connect_timeout 5s → 10s
2. `http_client.py`: 重试策略从 `HTTPError + TimeoutException` 改为只重试 `ConnectError + PoolTimeout`（ReadTimeout 不再重试，避免重复执行 Agent）
3. `message_handler.py`: AgentAPIError 和 unexpected Exception 时也调用 `_log_message()` 记录错误信息到 conversations 表

## ai_kefu: 添加对话上下文摘要功能
**问题**：Agent 每次 LLM 调用传入完整历史消息，但 Redis session 过期后上下文全部丢失；且长对话可能超出 token 限制。

**实现方案**：
1. 新建 `agent/context_summarizer.py`：
   - `should_summarize()`: 基于 token 估算判断是否需要摘要（阈值 4000 tokens）
   - `summarize_context()`: 调用 LLM 将旧消息生成 ≤300 字摘要，保留最近 4 条消息
   - `apply_summary_to_session()`: 将摘要存入 `session.context["context_summary"]`，并裁剪旧消息
2. `agent/types.py` AgentConfig 新增：`enable_context_summary`, `context_summary_token_threshold`, `context_summary_keep_recent`
3. `agent/executor.py`: 在 `run()` 开头加载 session 后检查是否需要摘要，触发则执行并保存
4. `agent/turn.py`: `execute_turn()` 中将 `session.context["context_summary"]` 注入到 system prompt 的末尾

## ai_kefu: 修复消息方向判断 + context 字段为空问题

**问题 1：conversations 表 context 字段全是 null**
- 根因：`message_handler._log_message()` 用 `hasattr(message, 'item_title')` 读取，但 `XianyuMessage` 模型没有 `item_title`/`item_price`/`message_id` 字段，Pydantic BaseModel 的 `hasattr` 永远返回 `False`
- 修复：
  - `models.py`: XianyuMessage 新增 `item_title`, `item_price`, `message_id`, `is_self_sent` 字段
  - `messaging_core.py`: `extract_message_data()` CHAT 分支提取 `reminderTitle` 和消息 ID 到 metadata
  - `run_xianyu.py`: 构造 XianyuMessage 时从 metadata 填充 `item_title`, `message_id`
  - `message_handler.py`: `_log_message()` 直接读 `message.item_title` 等字段，不再用 hasattr

**问题 2：没区分消息方向（自己的消息也触发 AI 回复）**
- 根因：`XianyuMessage.user_id` 是 `senderUserId`（发送者），系统不知道"我是谁"，WebSocket 会把自己发的消息同步回来
- 修复：
  - `config.py`: 新增 `seller_user_id` 配置
  - `run_xianyu.py`: 浏览器启动后自动从 Cookie `unb` 字段提取卖家 user_id
  - `run_xianyu.py`: 构造 XianyuMessage 时比较 `user_id == seller_user_id` 设置 `is_self_sent`
  - `message_handler.py`: `handle_message()` 中 `is_self_sent=True` 的消息只记录为 SELLER 类型，不触发 AI 回复
  - 历史消息也同理标记方向
