## Agent Turns LLM 输入输出持久化

在聊天记录查看器基础上，新增了 Agent 每轮 turn 的 LLM 输入/输出持久化和展示功能。

### 数据层
- **新建 `agent_turns` MySQL 表**：存储 session_id、turn_number、user_query、llm_input（完整 messages 数组 JSON）、llm_output（LLM 原始响应 JSON）、response_text、tool_calls、tool_results、duration_ms、success、error_message
- 在 `ConversationStore` 中新增 `save_turn()`、`get_turns_by_session()`、`get_recent_turns()` 方法

### Agent 执行层
- `TurnResult` (`agent/types.py`) 新增 `llm_input` 和 `llm_output` 字段
- `execute_turn()` (`agent/turn.py`) 在调用 LLM 前后捕获输入/输出快照
- `AgentExecutor` (`agent/executor.py`) 接受可选 `conversation_store` 参数，每轮 turn 后自动调用 `save_turn()` 持久化
- `chat.py` 路由在创建 executor 时注入 conversation_store

### API 层
- `GET /conversations/turns/recent` — 最近 turn 记录
- `GET /conversations/turns/{session_id}` — 某 session 的所有 turns
- `GET /conversations/{chat_id}` — 聊天详情（自动附带关联的 turns_by_session）

### 前端
- 聊天详情页新增 **"🔧 显示 Turns 详情"** 开关
- Turns 面板以可折叠卡片展示每轮 turn：Turn 编号、成功/失败状态、耗时、tool calls 数量
- 展开后可查看：用户查询、LLM 回复文本、Tool Calls（参数+结果）、LLM 输入 Messages 数组（可展开）、LLM 原始输出（可展开）
- JSON 数据用暗色代码块高亮显示
- 长文本内容支持弹窗查看完整内容
